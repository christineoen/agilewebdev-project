Part 1

HTML — Structure & Meaning

HTML (HyperText Markup Language) is the skeleton of every web page. It does not control how things look — that is CSS's job — but it defines what things are: headings, paragraphs, images, buttons, links. Browsers read this structure to build a Document Object Model (DOM), a live tree of objects that JavaScript can later inspect and modify.

Getting HTML right matters because search engines, screen readers, and other tools all rely on it to understand your content. A visually identical page can rank poorly in search or be inaccessible to assistive technology if the underlying markup is meaningless.

Document structure

Every HTML page shares the same outer shell. Understanding each piece is the first step to writing well-formed pages. 

<!-- Every HTML document starts with a doctype declaration -->
<!DOCTYPE html>
<html lang="en">

<head>
  <!-- charset tells the browser how to decode text -->
  <meta charset="UTF-8">

  <!-- viewport makes the page responsive on mobile devices -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- title appears in the browser tab and in search results -->
  <title>My Page</title>

  <!-- link external stylesheets here -->
  <link rel="stylesheet" href="styles.css">
</head>

<body>
  <!-- Everything visible goes here -->
  <h1>Hello, world!</h1>

  <!-- Scripts at the bottom so HTML loads before JS runs -->
  <script src="app.js"></script>
</body>
</html>

Common elements

HTML has over 100 elements, but a small handful covers the majority of real-world pages. Notice that elements carry meaning beyond their default appearance — a <button> is focusable and keyboard-activatable by default; a plain <div> styled to look like a button is not. 

<!-- Headings: h1 is the main topic, h2–h6 are subsections.
     Use only ONE h1 per page for good SEO. -->
<h1>Page title</h1>
<h2>Section heading</h2>

<!-- Paragraphs group blocks of text -->
<p>Some body text here.</p>

<!-- Anchor creates a hyperlink; href is the destination -->
<a href="about.html">About us</a>

<!-- Images: always include alt text for accessibility -->
<img src="photo.jpg" alt="A sunset over the mountains">

<!-- Lists: ul = unordered (bullets), ol = ordered (numbers) -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Button: use type="button" outside a form to avoid accidental submission -->
<button type="button">Click me</button>

Semantic HTML — why it matters

Before HTML5, developers wrapped everything in <div> and <span> tags, adding classes like class="header" to convey meaning. HTML5 introduced semantic elements — tags whose names describe their role in the page. 

<!-- Non-semantic: unclear intent -->
<div class="header">...</div>
<div class="nav">...</div>
<div class="main">
  <div class="article">...</div>
</div>
<div class="footer">...</div>
<!-- Semantic: intent is self-evident -->
<header>...</header>
<nav>...</nav>
<main>
  <article>...</article>
</main>
<footer>...</footer>

The semantic version is not just cleaner — screen readers announce landmarks like <nav> and <main> so keyboard users can jump directly to them, and search engines weight content inside <article> and <main> more highly than content in anonymous <div>s.

Part 2

CSS — Style & Layout

CSS (Cascading Style Sheets) controls the visual presentation of HTML: colours, fonts, spacing, layout, and animation. The "cascading" in the name describes how the browser resolves conflicts when multiple rules target the same element — a well-understood cascade is the difference between maintainable stylesheets and a patchwork of !important overrides.

Selectors & specificity

A CSS rule has two parts: a selector (which elements to style) and a declaration block (what styles to apply). When two rules conflict, the one with higher specificity wins. Understanding this prevents mysterious overrides. 

/* Element selector — lowest specificity, applies broadly */
p {
  color: #333;
  line-height: 1.6;
}

/* Class selector — more specific than element */
.highlight {
  background-color: yellow;
  font-weight: bold;
}

/* ID selector — more specific than class; use sparingly */
#hero-title {
  font-size: 3rem;
}

/* Compound: element INSIDE a class — good balance of specificity */
.card p {
  margin-bottom: 0;
}

/* Pseudo-class: targets element state, not the element itself */
a:hover {
  text-decoration: underline;
  color: #0056b3;
}
Best practice: Prefer class selectors over IDs in stylesheets. IDs are highly specific and hard to override without resorting to !important. Save IDs for JavaScript hooks (document.getElementById) or anchor links.

The box model

Every HTML element is a rectangular box made up of four layers. Misunderstanding the box model is the number-one cause of layout bugs.

margin
border
padding
content 
.box {
  /* content: width × height of the inner content area */
  width: 200px;
  height: 100px;

  /* padding: space inside the border, between content and border */
  padding: 16px;

  /* border: surrounds padding and content */
  border: 2px solid #333;

  /* margin: space outside the border, between this element and others */
  margin: 24px auto; /* top/bottom 24px, left/right auto (centres it) */

  /* box-sizing: border-box makes width include padding + border.
     Set this globally — it prevents countless layout surprises. */
  box-sizing: border-box;
}

Flexbox layout

Flexbox is a one-dimensional layout model that makes aligning and distributing items inside a container straightforward. It replaced older float-based hacks and is the foundation of most modern UI layouts (including Bootstrap's grid). 

/* The container controls how children are arranged */
.nav-bar {
  display: flex;
  flex-direction: row;           /* left → right (default) */
  align-items: center;         /* vertically centre children */
  justify-content: space-between; /* push logo left, links right */
  gap: 1rem;                   /* uniform space between children */
}

/* Children can grow/shrink to fill available space */
.search-input {
  flex: 1; /* grow to fill remaining width */
}

/* flex: 1 1 auto is shorthand for grow=1, shrink=1, basis=auto */
.sidebar {
  flex: 0 0 240px; /* fixed 240px — do not grow or shrink */
}

Part 3

JavaScript — Behaviour & Interactivity

JavaScript is the only programming language that browsers run natively. It transforms static HTML documents into dynamic, interactive applications — fetching data, responding to user actions, animating elements, and validating forms, all without a page reload.

Modern JavaScript (ES6+) is expressive and powerful. The key mental model is this: JavaScript reads and modifies the DOM that the browser built from your HTML. When you change the DOM, the browser immediately repaints what the user sees.

DOM manipulation

The DOM (Document Object Model) is the live, tree-shaped representation of your HTML page. JavaScript can query any node in the tree, read or change its content and attributes, and add or remove nodes entirely. 

// Select a single element by CSS selector (returns first match)
const title = document.querySelector('h1');

// Change text content — safe, never interprets HTML
title.textContent = 'Hello from JavaScript!';

// Change inline style directly
title.style.color = '#FBB04C';

// Add / remove CSS classes — preferred over inline styles
title.classList.add('highlight');
title.classList.remove('highlight');
title.classList.toggle('highlight'); // adds if absent, removes if present

// Read and set attributes
const link = document.querySelector('a');
console.log(link.getAttribute('href'));   // "/about.html"
link.setAttribute('href', '/contact.html');

// Create a new element and insert it into the page
const newItem = document.createElement('li');
newItem.textContent = 'New list item';
document.querySelector('ul').appendChild(newItem);

// Select multiple elements (returns a NodeList)
const allParagraphs = document.querySelectorAll('p');
allParagraphs.forEach(p => {
  p.style.lineHeight = '1.7';
});

Events

User interactions — clicks, key presses, form input, scroll — are communicated to JavaScript via events. You register an event listener on an element, and the browser calls your callback function each time the event fires. 

const btn = document.querySelector('#my-button');

// addEventListener(eventType, callback)
// The event object carries information about what happened.
btn.addEventListener('click', function(event) {
  console.log('Button clicked!', event.target);
});

// Arrow functions are common for concise callbacks
btn.addEventListener('click', (e) => {
  e.target.textContent = 'Clicked!';
  e.target.disabled = true;
});

// Respond to keyboard input on a text field
const input = document.querySelector('#search');
input.addEventListener('input', (e) => {
  // e.target.value holds the current text in the field
  console.log('User typed:', e.target.value);
});

// Event delegation: listen on a parent, handle children.
// More efficient than adding listeners to every child element.
document.querySelector('#item-list').addEventListener('click', (e) => {
  if (e.target.tagName === 'LI') {
    e.target.classList.toggle('done');
  }
});
